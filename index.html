ML Projects

  • Predicting self-cure to reduce recovery costs on delinquent customers

    The goal was to identify customers who would pay regardless of attempted collections calls and save resources. The project is now on the roadmap for delivery in 2023, thanks to collaboration with the Decision Science team. I initially attempted decision trees before moving onto XGBoost, as I learned that decision trees partition data into subsets based on feature values, while XGBoost uses decision trees and boosting techniques, therefore providing more accurate predictions.

  • Predicting House Prices: A Comprehensive End-to-End ML Project

    Looking to bridge the gap in my technical ability and Python experience, after being recommended O'Reilly's Hand's On Machine Learning with Scikit-Learn and Tensor Flow, I turned to a project in which I applied various machine learning techniques to predict house prices.

    The end-to-end machine learning project involved:

    • Data preprocessing
    • Exploratory data analysis
    • Feature engineering
    • Model selection
    • Hyperparameter tuning

    For this project, I utilised decision trees and random forests to predict house prices using machine learning techniques. I learned multiple elements of the ML process, best practices as well as how to ensure that the models were accurate and could be applied to new data, I evaluated them using mean squared error and cross-validation techniques. The results showed that the random forest model outperformed the decision tree model and was selected to make predictions on new data.

  • Optimising Call Center Performance by Identifying the Best Time to Dial using SQL and Segmentation Analysis

    While this isn't strictly ML, it's still worth mentioning as part of my journey to become a data scientist. I worked on a project to optimise call center dialler operations by analysing RPC performance of different segments based on various features. While many were considered, including credit bureau and multiple internal data points, when it came to RPC by hour of the day, age and gender had the biggest impact. People at similar life stages are more likely to have similar routines, such as the elderly being available early morning, compared to young parents who are likely unavailable during school runs.

    The project involved the following steps:

    • Data collection and preprocessing
    • Feature engineering and selection
    • segmentation Performance analysis
    • Implementation involved predicting staff availability, and distribution based on segment performance for each hour

    The results of this project showed that the "model" was able to accurately predict the RPC rate by age and gender group for each hour of the day, thus enabling the call center to optimise its dialling operations and increase customer contact rates.

  • Searching for Credit Card Numbers in SQL Databases with Python

    This project involved searching for 16-digit credit card numbers in SQL databases using Python. Certain tables are excluded, and the results are inserted into a designated table. The script uses the Python modules pyodbc and concurrent.futures to connect to the databases and perform the search concurrently. Any errors encountered are caught and inserted into the designated table, along with the status of the operation and the error message. The script outputs a message for each database searched and prints any errors encountered.
