ML Projects

  • Predicting self-cure to reduce recovery costs on delinquent customers

    The goal was to identify customers who would pay regardless of attempted collections calls and save resources. The project is now on the roadmap for delivery in 2023, thanks to collaboration with the Decision Science team. I initially attempted decision trees before moving onto XGBoost, as I learned that decision trees partition data into subsets based on feature values, while XGBoost uses decision trees and boosting techniques, therefore providing more accurate predictions.

  • Predicting House Prices using Machine Learning Techniques: A Comprehensive End-to-End Project

    Looking to bridge the gap in my technical ability and Python experience, after being recommended O'Reilly's Hand's On Machine Learning with Scikit-Learn and Tensor Flow, I turned to a project in which I applied various machine learning techniques to predict house prices.

    The end-to-end machine learning project involved:

    • Data preprocessing
    • Exploratory data analysis
    • Feature engineering
    • Model selection
    • Hyperparameter tuning

    For this project, I utilized decision trees and random forests to predict house prices using machine learning techniques. I learned multiple elements of the ML process, best practices as well as how to ensure that the models were accurate and could be applied to new data, I evaluated them using mean squared error and cross-validation techniques. The results showed that the random forest model outperformed the decision tree model and was selected to make predictions on new data.

  • Optimising Call Center Performance by Identifying the Best Time to Dial using SQL and Segmentation Analysis

    While this isn't strictly ML, the approach leveraged multiple concepts which are worth mentioned as part of my journey to become a Machine Learning Engineer, I worked on a project to optimise call center operations using machine learning techniques. The goal of this project was to identify the best time to dial by hour of day, age, and gender using internal data and SQL. I utilized machine learning algorithms to build a model that could predict the right party contact (RPC) rate for each hour of the day based on historical data.

    The project involved the following steps:

    • Data collection and preprocessing
    • SQL database design and implementation
    • Feature engineering and selection
    • Evaluation using various metrics

    The results of this project showed that the "model" was able to accurately predict the RPC rate by age and gender group for each hour of the day, thus enabling the call center to optimise its dialling operations and increase customer contact rates.
